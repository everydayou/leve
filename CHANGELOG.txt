ROUND 53 — SHEET & CTA POLISH (2026-06-13)
================================================================
Files: Sheet.tsx, AddEntrySheet.tsx, WeightLogSheet.tsx

GRADIENT REACHING SCREEN BOTTOM:
  Root cause: Sheet scroll area had safe-bottom class (~34px padding).
    sticky CTA stopped at top of safe zone, not screen edge.
  Fix: removed safe-bottom from scroll area. All sticky CTA divs use inline
    paddingBottom: 'max(1rem, env(safe-area-inset-bottom, 0px))'.

SOURCE PICKER FLASH FIX:
  Root cause: setShowSourcePicker(false) called BEFORE awaiting camera/library.
    Default FoodForm content flashed briefly.
  Fix: setShowSourcePicker(false) moved to AFTER photo returned or error.
  Source picker stays visible while native OS UI is open.
  Cancel keeps source picker visible; user can retry or dismiss.

DISABLED BUTTON GRADIENT FIX:
  Root cause: gradient reached transparent at button position (75% transparent
    at ~22-26% into the sticky div). Disabled button (opacity-40) on 75% transparent
    background let scroll content show through.
  Fix: gradient stops at 1.5rem (= pt-6 top padding):
    linear-gradient(to bottom, transparent 0px, var(--color-surface) 1.5rem)
  Button area always on solid surface.

SHEET COLLAPSE LOGIC FIX:
  Old order was wrong: dismiss check fired BEFORE collapse check.
  New order:
    1. dragY < -EXPAND_THRESHOLD && !expanded && !forceExpanded → expand to 90%
    2. expanded && !forceExpanded && dragY > 60 → collapse to 50% (always)
    3. dragY > DRAG_DISMISS_PX && (!expanded || forceExpanded) → dismiss
    4. else → spring back
  From 90%: any downward drag > 60px collapses to 50% first.
  From 50%: downward drag > 110px dismisses.
  forceExpanded sheets dismiss directly.

  Easing history: cubic-bezier(0.32, 0.72, 0, 1) → cubic-bezier(0.16, 1, 0.3, 1)
    → final: cubic-bezier(0.22, 0.65, 0.25, 1) (smooth even deceleration).
  max-height fix: was toggling 'auto'/undefined (CSS can't interpolate).
    Now always a concrete value: '50dvh' (collapsed) or '90dvh' (expanded).
