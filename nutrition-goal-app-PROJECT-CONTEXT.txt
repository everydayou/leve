0B. DESIGN-SYSTEM CONTRACT — FIXED RULES (read BEFORE editing any UI)
----------------------------------------------------------------
Non-negotiable standing rules. Any chat touching src/ui MUST follow them.

1. COMPONENTS FIRST. Build UI from kit atoms in src/ui/kit: Button/IconButton,
   Sheet, LabeledInput, NumberField, Card/Surface, StatTile, ProgressBar/
   GaugeArc, Badge, SegmentedControl, FilterPills, Icon, etc. NEVER hand-roll
   a button/input/sheet/stat inline when a kit atom exists.
2. TOKENS ONLY. Colour, radius, shadow and type from src/index.css semantic
   tokens. NEVER raw Tailwind text sizes (text-xs/sm/base/lg/xl) — use
   text-caption/footnote/subhead/body/callout/headline/title/display/hero.
   NEVER hardcode hex/rgb — use colour tokens.
3. ICONS, NOT GLYPHS. Use <Icon> for all affordances. Never type ›, ‹ or ×.
4. ONE SOURCE OF TRUTH. Change look via token or atom, never per-screen CSS.
5. ENFORCEMENT. eslint.config.js BLOCKS raw text sizes and hex/rgb in src/ui.
   Run `npm run lint` before shipping. Do NOT disable guardrails.
6. ACCESSIBILITY FIRST. Focus outlines, ARIA roles, prefersReducedMotion(),
   WCAG AA contrast, 44×44pt touch targets, labeled form inputs.

Text scale: caption(12) / footnote(13) / subhead(15) / callout(16) /
headline(18) / title(22) / display(30) / hero(48).
WHERE THINGS LIVE: tokens = src/index.css (@theme); atoms = src/ui/kit/;
live reference = /styleguide route.

----------------------------------------------------------------
1. WHAT WE'RE BUILDING
----------------------------------------------------------------
A personal nutrition + weight-management app for Marco's own use.
NOT a calorie tracker. Core idea: GOAL MANAGEMENT — tracking exists only
to answer "Am I on track to hit my goal?"
Mental model: Goal → Weekly progress → Daily progress → Food/Activity inputs.

----------------------------------------------------------------
2. PLATFORM & STACK
----------------------------------------------------------------
React + Capacitor (web code + native iOS shell). NOT native Swift.
Stack: Vite 6, React 19, TypeScript, Tailwind CSS v4, Dexie (IndexedDB),
Vitest. Capacitor v8. appId: com.marco.nutritiongoal, webDir: dist.
Install path: Xcode + free Apple ID ($0). Re-sign every ~7 days.

----------------------------------------------------------------
3. AGREED V1 SCOPE
----------------------------------------------------------------
- Goal types: lose_by_date and gain_by_date.
- TODAY: frequent foods, Add-entry sheet, calories + protein, deficit,
  swipeable day nav.
- GOAL: weight trend vs. target line, 9 lifecycle states.
- PANTRY: create/edit food items.
- Manual meal entry, manual activity calories, weight logging.
- BMR: single user-entered number in V1.
- Account: weight history, profile, BMR, export (JSON).
DEFER (V1.5+): BMR auto-calibration, HealthKit/Withings, History tab,
per-piece measurement types.

----------------------------------------------------------------

  Scan item only shown when VITE_FOOD_SCAN_API_URL is set (SCAN_ENABLED gate).
- Add-entry = bottom sheet (never full-screen nav).
- Goal Setup = full-screen flow. ?skip-type=true skips plan-type step.
- Day nav on Today: chevrons + swipe. Future dates disabled.

----------------------------------------------------------------
4B. V1 DATA MODEL
----------------------------------------------------------------
Goal status: active / completed / abandoned.
GoalScreen renders 9 states (all implemented r55):
  (1) no-goal  (2) active-normal  (3) final-day·no-weight
  (4) final-day·weight-in(not hit)  (5) final-day·goal-hit
  (6) early-complete  (7) overdue  (8) completed  (9) abandoned/ended
GoalOutcomeView: createPortal → document.body. Slide-up on mount.
GoalSettingsSheet (⋯): Mark complete / Edit plan / End goal / Past goals.
User model: weightCadence ('daily'|'weekly'), weeklyWeightDay.

----------------------------------------------------------------
5. METRICS / MATH
----------------------------------------------------------------
Total Burn = BMR + Active Calories.
Daily Deficit = Total Burn − Calories Consumed.
~7700 kcal per kg for goal pace math.

8. ARCHITECTURE NOTES
----------------------------------------------------------------
Layer model (strict dependency direction):
  src/domain/   Pure TS — types.ts, calc.ts, goal.ts, bmr.ts.
  src/data/     Dexie + memory repos. repositories.ts = interfaces.
  src/state/    repos.ts (swap point) + useLive() + useDay().
  src/ui/       AppShell + screens + components + kit/.
  src/index.css Design token layer (Tailwind v4 @theme).

KEY PATTERNS:
- createPortal: GoalOutcomeView renders into document.body to escape
  AppShell overflow-hidden / z-index on WKWebView.
- slide-up/down: .slide-up-in / .slide-down-out in index.css.
  Exit: setExiting(true) → setTimeout(cb, 320) → navigate.
- Sheet-on-Sheet: destructive confirms render a second Sheet alongside
  the parent in a Fragment (both conditionally mounted).
- SCAN_ENABLED = !!(import.meta.env.VITE_FOOD_SCAN_API_URL) — used in
  AddEntrySheet.tsx header icon and FloatingTabBar.tsx FAB action list.

ENV NOTE: Cowork Linux sandbox can't run rollup native binary. After
changes, on the Mac: `npm install` → `npm test` → `npm run build` →
`npm run ios:sync`.
